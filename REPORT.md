# Отчет по практической работе 13

## 1. Название практической работы

Практическая работа 13. Создание простого приложения на React.

## 2. Цель работы

Изучить основы разработки простого React-приложения с помощью Vite и закрепить работу с компонентами, props, состоянием, эффектами, событиями, условным рендерингом, списками и сохранением данных в браузере.

## 3. Используемые технологии

- Vite
- React
- JavaScript
- CSS
- localStorage
- Git
- GitHub Pages

## 4. Описание реализованного приложения

Разработано приложение «Менеджер задач». Пользователь может добавлять новые задачи, отмечать их выполненными, удалять ненужные задачи и переключать отображение списка между всеми, активными и выполненными задачами. Интерфейс выполнен в виде аккуратной адаптивной панели с карточками задач.

## 5. Реализованный функционал

- Добавление новой задачи через форму ввода.
- Проверка пустого значения при добавлении.
- Отметка задачи как выполненной или активной.
- Удаление задачи из списка.
- Фильтрация задач: все, активные, выполненные.
- Подсчет общего количества задач, активных задач и выполненных задач.
- Отображение счетчика оставшихся активных задач.
- Сохранение списка задач в `localStorage`.
- Восстановление задач после обновления страницы.
- Условное отображение сообщения, если по выбранному фильтру задач нет.
- Адаптивная верстка для мобильных экранов.

## 7. Основные React-концепции, использованные в работе

- Компоненты: приложение разделено на `App`, `TodoForm`, `TodoItem`, `TodoFilter` и `TodoStats`.
- Props: данные и функции передаются из родительского компонента в дочерние.
- `useState`: используется для хранения списка задач, выбранного фильтра, текста формы и ошибки ввода.
- `useEffect`: используется для сохранения задач в `localStorage` при изменении списка.
- Обработка событий: реализованы события отправки формы, изменения поля ввода, нажатия кнопок и переключения checkbox.
- Условный рендеринг: отображается сообщение о пустом списке, а также ошибка при пустом вводе.
- Рендеринг списков через `map`: задачи и кнопки фильтра создаются на основе массивов.

## 8. Работа с localStorage

Для хранения задач используется ключ `practice_13_react_todos`. При первом открытии приложение показывает стартовые задачи. Если в `localStorage` уже есть сохраненный список, приложение читает его при инициализации состояния. После каждого изменения списка `useEffect` записывает актуальные задачи обратно в `localStorage`, поэтому после обновления страницы данные сохраняются.

## 9. Инструкция по запуску проекта локально

1. Перейти в папку проекта:

```bash
cd practice-13-react
```

2. Установить зависимости:

```bash
npm install
```

3. Запустить проект:

```bash
npm run dev
```

4. Открыть локальный адрес, который выведет Vite.

## 10. Инструкция по сборке проекта

```bash
npm run build
```

После успешной сборки готовые файлы будут находиться в папке `dist`.

## 11. Инструкция по публикации на GitHub Pages

1. Создать или открыть репозиторий `git21git/practice_13_react`.
2. Добавить удаленный репозиторий:

```bash
git remote add origin https://github.com/git21git/practice_13_react.git
```

3. Зафиксировать изменения и отправить проект в ветку `main`:

```bash
git add .
git commit -m "Complete practice 13 React todo manager"
git push -u origin main
```

4. Опубликовать приложение:

```bash
npm run deploy
```

5. В настройках GitHub Pages выбрать источник публикации: `gh-pages`, папка `/root`.
